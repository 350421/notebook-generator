## 17. 2026-07-06 第二轮更新：分页稳定性打磨

本轮只修改了 enderer.py 一个源码文件（约 65 行增量），不改数据模型、不改 API、不改前端。

### 17.1 长列表可跨页拆分 (_split_long_lists)

**问题**：超过 3 条的列表块是"原子级"的——分页器把它当成不可分割的单一块。长列表要么全塞进当前页（可能溢出），要么整体推到下一页（导致上一页留白过大）。

**做法**：在 ender_template() 中分页测量之前，新增 _split_long_lists() 函数，将超过 3 条的列表块按 2-3 条一组拆分为多个子块。有序列表子块额外携带 list_start 字段，_render_list() 据此输出 HTML start 属性延续编号。

**关键设计**：
- source_indices 同步扩展：同一原始列表的所有子块指向同一个原始索引
- lock_start / lock_end / 图片插入定位不受影响
- 小标题跨页连续编号（基于原始 blocks 数组遍历）完全不受影响

### 17.2 最后一页稀疏检测

**问题**：贪心分页算法最后一页自然接收"剩下的所有内容"，如果只剩一个短段落，会独占一整页，留白高达 60-80%。

**做法**：在 _paginate() 返回 pages 后检测最后一页内容占比。若不足可用高度 30% 且总页数 >= 2，尝试从倒数第二页末尾回移 1 个 block。

### 17.3 分页失败错误提示产品化

**问题**：失败时只提示"存在无法完整放入单页的超长段落"，用户不知道具体是哪个块。

**做法**：遍历测量数据，找出超过可用高度的块，用 _block_summary() 生成可读描述，指明块类型+内容摘要+实际高度+可用高度。

## 18. 2026-07-06 第三轮更新：修复标题检测 + 封面联动

### 18.1 问题根因

用户报告两个问题：
1. 粘贴文案后不会自动生成标题
2. 启用封面模式后没有效果，每次改设置都要手动点「重新生成预览」

两个问题的根因是同一个：markdown_parser.py 的 _smart_heading_indices() 对纯文本标题的识别阈值过严——len(first_text) <= 30。真实场景中常见的标题（如「2026年最值得做的5个副业方向，从零到月入过万的完整实操攻略」31 字）会被漏检为 ody。

没有 title block → enderer.py 的 _extract_cover_page() 直接返回 None → 封面模式完全无效果 → 用户误以为 auto-sync 不工作。

### 18.2 修改内容

| 文件 | 改动 |
|------|------|
| markdown_parser.py:120 | len(first_text) <= 30 → <= 50 |
| markdown_parser.py:146 | len(text) > 15 → > 22（小标题检测） |
| static/script.js:551-554 | auto-sync 无法触发时的提示改为「请先点击生成预览」 |

### 18.3 验证结果

- 31 字标题 → ✅ 正确识别为 title
- 45 字标题 → ✅ 正确识别为 title
- 31 字标题 + 封面启用 → ✅ 封面页正确渲染（data-page-mode="cover"）
